# RAG Web UI 重构 - 需求分析

**项目名称**: RAG Knowledge Base Web UI  
**负责人**: MC  
**版本**: v1.0  
**日期**: 2026-03-01

---

## 一、项目背景与目标

### 1.1 背景
RAG知识库系统后端已稳定运行，但现有的 Vue3 Web UI 存在诸多问题：
- 代码结构混乱，维护困难
- 文件上传功能不稳定
- 状态管理混乱
- 用户体验差

**决定**: 废弃现有前端，重新开发。

### 1.2 目标
构建一个 **现代化、稳定、易用** 的 RAG Web UI：
- 清晰直观的项目管理和文档操作
- 流畅的文件上传体验
- 高效的语义搜索界面
- 响应式设计，支持多端访问

## 四、非功能需求

### 4.1 性能要求
- 首屏加载 < 3秒
- 搜索响应 < 2秒
- 文件上传显示实时进度

### 4.2 兼容性要求
- Chrome/Edge/Safari/Firefox 最新2个版本
- 响应式：桌面端(1280+)、平板(768+)、手机(375+)

### 4.3 安全要求
- JWT Token 过期自动跳转登录
- 401 错误统一处理

## 六、参考与附件

### 后端 API 文档
- Base URL: `/api/v1`
- 认证: Bearer Token
- 完整 API 见: `src/rag_api/routers/`

### 核心 API 列表

| 接口 | 方法 | 路径 | 功能 |
|-----|------|------|------|
| 登录 | POST | `/api/v1/auth/login/json` | 获取JWT |
| 项目列表 | GET | `/api/v1/projects` | 获取所有项目 |
| 创建项目 | POST | `/api/v1/projects` | 创建新项目 |
| 更新项目 | PUT | `/api/v1/projects/{id}` | 更新项目信息 |
| 删除项目 | DELETE | `/api/v1/projects/{id}` | 删除项目 |
| 文档列表 | GET | `/api/v1/projects/{id}/documents` | 获取项目文档 |
| 上传文档 | POST | `/api/v1/projects/{id}/documents` | 上传单个文档 |
| 删除文档 | DELETE | `/api/v1/projects/{id}/documents/{doc_id}` | 删除文档 |
| 搜索 | POST | `/api/v1/search` | 语义/关键词搜索 |
| Watcher状态 | GET | `/api/v1/watcher/status` | 获取监控状态 |
| Watcher启停 | POST | `/api/v1/watcher/start|stop` | 控制监控 |

### 现有后端结构
```
src/rag_api/
├── main.py          # FastAPI入口
├── routers/
│   ├── auth.py      # 认证
│   ├── projects.py  # 项目管理
│   ├── documents.py # 文档管理
│   ├── search.py    # 搜索
│   └── watcher.py   # 文件监控
```
